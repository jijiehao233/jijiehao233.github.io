基本语法

  1. SQL对大小写不敏感；
  2. 某些数据库系统要求在每条 SQL 语句的末端使用分号，分号是在数据库系统中分隔每条 SQL 语句的标准方法，这样就可以在对服务器的相同请求中执行一条以上的 SQL 语句；
  3. 注释
    单行注释：句首使用 "-- "
    多行注释：使用/*... */
  4. SELECT 语句的执行顺序
    from语句 > where语句(结合条件) > start with语句 > connect by语句 > where语句 > group by语句 > having语句 > model语句 > select语句 > union、minus、intersect等集合演算演算 > order by语句

命令语句

SELECT - 从数据库中提取数据
UPDATE - 更新数据库中的数据
DELETE - 从数据库中删除数据
INSERT INTO - 向数据库中插入新数据
CREATE DATABASE - 创建新数据库
ALTER DATABASE - 修改数据库

ALTER TABLE - 变更（改变）数据库表
DROP TABLE - 删除表
CREATE INDEX - 创建索引（搜索键）
DROP INDEX - 删除索引

create创建

  1. 创建数据库
    CREATE DATABASE dbname;

  2. 创建表格
    CREATE TABLE table_name
    (
    column_name1 data_type(size) NOT NULL PRIMARY KEY,
    column_name2 data_type(size) NOT NULL,
    column_name3 data_type(size),
    ALTER TABLE table_name ADD CONSTRAINT pk_命名 PRIMARY KEY (column_name1,column_name2)；
    ....
    );

insert插入

  1. 不指定列名
    INSERT INTO table_name
    VALUES (value1,value2,value3,...);

  2. 制定列名
    INSERT INTO table_name (column1,column2,column3,...)
    VALUES
    (value11,value21,value31,...);
    (value12,value22,value33,...);

alter

  1. 添加列
    ALTER TABLE table_name ADD column_name datatype
  2. 将column_name1移动到column_name2后面
    ALTER TABLE table_name MODIFY column_name1 datatype AFTER column_name2

join连接

  1. 语法
    select * form t1 left join t2 on (t1. column_name = t2. column_name) where t2. column_name = '值'
  2. 注意
    • 条件on 和 where
      on 条件是在生成临时表时使用的条件，它不管 on 中的条件是否为真，都会返回left或right表中 记录，另一边看是否满足on的条件；
      where 条件是在临时表生成好后，再对临时表进行过滤的条件，这时已经没有 join 的含义了，条件不为真的就全部过滤掉；
      left join,right join,full join的特殊性，不管on上的条件是否为真都会返回left或right表中的记录，full则具有left和right的特性的并集;
      而inner jion没这个特殊性，则条件放在on中和where中，返回的结果集是相同的

数据类型

String类型

数据类型 描述
Bit 允许 0、1 或 Null
Character(N) 字符/字符串。固定长度 N
Nchar(N) 固定长度的 Unicode 字符串。最多 4,000 个字符
Ntext 可变长度的 Unicode 字符串。最多 2Gb 文本数据
Nvarchar 可变长度的 Unicode 字符串。最多 4,000 个字符
Nvarchar(Max) 可变长度的 Unicode 字符串。最多 536,870,912 个字符。
Text 可变长度的字符串。最多 2Gb 文本数据。
Varchar(N) 或character Varying(N) 字符/字符串。可变长度。最大长度 N
Ntext 可变长度的 Unicode 字符串。最多 2Gb 文本数据。
Image 可变长度的二进制字符串。最多 2Gb。

Number 类型

数据类型 描述
Bigint 整数值（没有小数点）。精度 19
Decimal(P,S) 精确数值，精度 P，小数点后位数 S。例如：Decimal(5,2) 是一个小数点前有 3 位数，小数点后有 2 位数的数字
Double Precision 近似数值，尾数精度 16
Float(P) 近似数值，尾数精度 P。一个采用以 10 为基数的指数计数法的浮点数。该类型的 Size 参数由一个指定最小精度的单一数字组成
Integer(P) 整数值（没有小数点）。精度 P
Money 介于 -922,337,203,685,477.5808 与 922,337,203,685,477.5807 之间的货币数据
Numeric(P,S) 精确数值，精度 P，小数点后位数 S。（与 Decimal 相同）
Real 近似数值，尾数精度 7
Smallint 整数值（没有小数点）。精度 5
Smallmoney 介于 -214,748.3648 与 214,748.3647 之间的货币数据
Tinyint 允许从 0 到 255 的所有数字

Date 类型

数据类型 描述
Date 存储年、月、日的值
Datetime 从 1753 年 1 月 1 日 到 9999 年 12 月 31 日，精度为 3.33 毫秒
Datetime2 从 1753 年 1 月 1 日 到 9999 年 12 月 31 日，精度为 100 纳秒
Datetimeoffset 与 Datetime2 相同，外加时区偏移
Interval 由一些整数字段组成，代表一段时间，取决于区间的类型
Smalldatetime 从 1900 年 1 月 1 日 到 2079 年 6 月 6 日，精度为 1 分钟
Time 存储小时、分、秒的值
Timestamp 存储年、月、日、小时、分、秒的值

其它类型

数据类型 描述
Array 元素的固定长度的有序集合
Binary(N) 二进制串。固定长度 N
Boolean 存储 True 或 False 值
Cursor 存储对用于数据库操作的指针的引用
Multiset 元素的可变长度的无序集合
Sql_Variant 存储最多 8,000 字节不同数据类型的数据，除了 Text、Ntext 以及 Timestamp
Table 存储结果集，供稍后处理
Uniqueidentifier 存储全局唯一标识符 (Guid)
Varbinary(N) 或binary Varying(N) 二进制串。可变长度。最大长度 N
Xml 存储 Xml 数据
